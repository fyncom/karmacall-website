## Behind the Scenes

Here's what happens when you save an MDX file:

1. **Frontmatter Parsing**: The converter extracts metadata (title, description, etc.)
2. **Content Conversion**: Markdown is converted to JSX with proper styling
3. **Component Integration**: Your content is wrapped in our blog components
4. **Database Updates**: Article listings are automatically updated
5. **SEO Optimization**: Meta tags and social sharing are configured

### Frontmatter Requirements

Every MDX article needs these frontmatter fields:

- `title`: Your article title (appears in browser tab and social shares)
- `description`: SEO description (150-160 characters optimal)
- `author`: Author name
- `date`: Publication date (YYYY-MM-DD format)
- `slug`: URL path (must start with `/blog/`)

Optional fields:
- `featuredImage`: Path to hero image for social sharing
- `keywords`: Array of SEO keywords

## Getting Started

To create a new blog post:

1. Create a `.mdx` file in the `blog-posts/` directory
2. Add proper frontmatter at the top
3. Write your content in Markdown
4. Save the file - conversion happens automatically in watch mode!

The converter will:
- Generate a JavaScript file in `src/pages/blog/`
- Update the blog index
- Add the article to the similarity database
- Optimize for social sharing
